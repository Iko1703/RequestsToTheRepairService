# PROMPTS.md — Промпты, использованные при разработке

## Основные промпты для AI-ассистента

### 1. Первоначальное планирование
```
Я разрабатываю небольшое веб-приложение для приёма и обработки заявок в ремонтную службу. 
Это является тестовым заданием в компанию. Из обязательного функционала важно иметь:
- Роли: диспетчер и мастер (простая авторизация)
- Заявка с полями: clientName, phone, address, problemText, status, assignedTo, createdAt, updatedAt
- Страницы: создание заявки, панель диспетчера, панель мастера
- Важно проверять гонку запросов
- Два варианта запуска: Docker и без него

Для начала хочу разобраться со стеком: Spring Boot, PostgreSQL, JPA/Hibernate, REST + HTML формы, Spring Security, Docker Compose.
Распиши подробно весь ход работы по этапам.
```

### 2. Модель данных и архитектура
```
Давай начнем с этапа модели данных. На мой взгляд нам необходимы: User, Ticket (с обязательными полями), TicketEvent.
Также сразу проработаем слои (controller, service, repository, model, security, dto), эндпоинты.
```

### 3. Безопасность и race conditions
```
Идем дальше, сейчас разберем безопасность и race conditions. 
Я думаю лучше использовать Optimistic locking в данном проекте.
```

### 4. CRUD и UI
```
Делаем реализацию CRUD и бизнес-логики, также UI.
```

### 5. Аудит, обработка ошибок, тестирование
```
Сейчас разберем audit log, обработку ошибок как бэка так и фронта, и тестирование (важно Unit и на гонку).
```

### 6. Docker

Docker и запуск — наш дальнейший пункт.
```

### 7. Исправление тестов и финализация
```
Мне нужны зеленые тесты, а еще важно, чтобы работала БД.

## Итоговый результат

В результате использования AI-ассистента было создано:
- Полноценное Spring Boot приложение с REST API и Thymeleaf UI
- JPA-сущности с optimistic locking для предотвращения гонок
- Spring Security с ролевой моделью (DISPATCHER, MASTER)
- Интеграционные тесты, включая тест на race condition
- Docker Compose конфигурация для развёртывания
- Сиды данных (data.sql) для БД
